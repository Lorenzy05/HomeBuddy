# 🏠 HomeBuddy 项目分部分完成方案

---

## 📊 总览：5 个部分

| 部分 | 名称 | 核心产出 | 预计工时 |
|:---:|------|---------|:-------:|
| **Part 1** | 🔩 硬件平台搭建 | 机器人能通电、能推着走 | 中 |
| **Part 2** | ⚙️ 执行器固件开发 | 所有硬件能被串口指令控制 | 中 |
| **Part 3** | 🎮 遥控控制 | 遥控器能控制移动 + 云台 | 中 |
| **Part 4** | 🧠 树莓派 AI 核心 | 人脸检测 + 语音交互 | 大 |
| **Part 5** | 🎯 系统整合与场景 | 完整功能闭环（送货 + 跟随） | 大 |

## Part 1 🔩 硬件平台搭建

### 🎯 目标

让机器人有一个**稳定的物理平台**，所有硬件固定好，能通电，能用手推着走。

### 📦 任务清单

| # | 任务 | 说明 |
|:-:|------|------|
| 1.1 | 🛞 底盘制作 | 购买亚克力板或 3D 打印，30×25cm，标记孔位 |
| 1.2 | 🔩 电机 + 轮子安装 | 固定 L298N、2 个直流电机、轮子、万向轮 |
| 1.3 | 🧱 分层堆叠 | 铜柱分层：底层（电池+L298N）、中层（ESP32）、上层（树莓派） |
| 1.4 | 📷 摄像头 + 云台 | USB Webcam 固定在 SG90 舵机云台上 |
| 1.5 | 🔌 所有硬件固定 | 用扎带 + 纳米胶固定所有模块 |
| 1.6 | 🔋 供电系统测试 | 移动电源 → 树莓派 + ESP32；锂电池 → L298N |
| 1.7 | 🧪 通电验证 | 树莓派绿灯亮、ESP32 灯亮、L298N 灯亮 |

### ✅ 完成标准

- [ ] 机器人放在地上能站稳，不晃动
- [ ] 用手推着轮子能顺畅转动
- [ ] 所有板子通电正常（LED 亮）
- [ ] 所有线缆整齐，没有松动

---

## Part 2 ⚙️ 执行器固件开发

### 🎯 目标

ESP32 能够**接收串口指令**并控制所有硬件（电机、舵机、OLED、NeoPixel、MPU6050）。

### 📦 任务清单

| # | 任务 | 说明 |
|:-:|------|------|
| 2.1 | 📡 串口通信基础 | 树莓派 ↔ ESP32 UART 连接，收发测试 |
| 2.2 | 🚗 电机控制 | 解析 `!MOTOR#L{左}#R{右}#END` → L298N PWM 输出 |
| 2.3 | 🦾 舵机云台控制 | 解析 `!PAN` / `!TILT` → SG90 角度控制 |
| 2.4 | 🖥️ OLED 表情 | 解析 `!EMOJI` → 显示 😊 🎯 😢 🤖 |
| 2.5 | 💡 NeoPixel 灯效 | 解析 `!RGB` → 显示不同颜色 |
| 2.6 | 🧭 MPU6050 读取 | 每 100ms 发送 `!IMU#ROLL#PITCH#END` 给树莓派 |
| 2.7 | 🔋 ADC 电量监测 | 读取电池电压，发送 `!BATTERY` |

### ✅ 完成标准

- [ ] 串口发 `!MOTOR#L50#R50#END` → 轮子正转
- [ ] 串口发 `!MOTOR#L-50#R-50#END` → 轮子反转
- [ ] 串口发 `!PAN#30#END` → 舵机转到对应角度
- [ ] 串口发 `!EMOJI#1#END` → OLED 显示 🎯
- [ ] 串口持续收到 `!IMU` 数据
- [ ] 所有指令响应时间 < 50ms

---

## Part 3 🎮 遥控控制

### 🎯 目标

红外遥控器能**控制机器人移动和云台**，作为最可靠的手动控制方式。

### 📦 任务清单

| # | 任务 | 说明 |
|:-:|------|------|
| 3.1 | 📟 HX1838 解码 | 烧录 IRrecv 示例，读取每个按键的编码 |
| 3.2 | 🗺️ 按键映射表 | 建立按键编码 → 功能（前进/后退/左转/右转/停止/云台） |
| 3.3 | 🚗 方向键控制 | ↑↓←→ 控制移动，OK 紧急停止 |
| 3.4 | 🔄 数字键控制 | 1 回中，2-5 云台微调，0 模式切换 |
| 3.5 | ⏩ 速度调节 | * 加速，# 减速，3 档速度可调 |
| 3.6 | 📟 遥控优先级 | 按下方向键 → 立即暂停所有自动模式 |
| 3.7 | 🔄 模式切换 | 遥控模式 ↔ 跟随模式 ↔ 送货模式 |

### ✅ 完成标准

- [ ] 按下 ↑ → 前进，松开 → 停止
- [ ] 按下 ← → 原地左转，松开 → 停止
- [ ] 按下 OK → 立即紧急停止
- [ ] 按 2 → 云台左转，按 3 → 云台右转
- [ ] 按 0 → 循环切换三种模式
- [ ] 跟随模式下按方向键 → 立即切换到遥控模式

## Part 4 🧠 树莓派 AI 核心

### 🎯 目标

树莓派实现**人脸检测、语音识别、语音合成、智能对话**四大 AI 能力。

### 📦 任务清单

| # | 任务 | 说明 |
|:-:|------|------|
| 4.1 | 📷 USB Webcam 测试 | `libcamera` / `cv2.VideoCapture(0)` 画面正常 |
| 4.2 | 👁️ MediaPipe 人脸检测 | 画面中框出人脸，输出人脸中心坐标 (cx, cy) |
| 4.3 | 🎤 USB 麦克风测试 | `arecord` 录音测试，确认设备号 |
| 4.4 | 🎤 Vosk STT 集成 | 录音 → 识别 → 输出中文文字 |
| 4.5 | 🔊 USB 音箱测试 | `aplay` 播放测试，确认设备号 |
| 4.6 | 🔊 Edge-TTS 集成 | 文字 → 音频 → 音箱播放 |
| 4.7 | 🤖 Ollama 集成 | 安装 Ollama，下载 Gemma 2B，测试对话 |
| 4.8 | 🎯 人脸跟随算法 | 人脸偏移 → PID 控制 → 生成速度指令 |

### ✅ 完成标准

- [ ] OpenCV 窗口显示摄像头画面
- [ ] 画面中你的人脸被绿色框框住
- [ ] 说"你好" → 终端输出"你好"文字
- [ ] 树莓派说"测试语音" → 音箱听到声音
- [ ] 问"今天天气怎么样" → LLM 返回回答
- [ ] 人脸左右移动 → 终端输出左右速度指令

## Part 5 🎯 系统整合与场景

### 🎯 目标

所有模块整合，实现**完整的用户场景**：语音对话、遥控控制、人脸跟随、自动送货。

### 📦 任务清单

| # | 任务 | 说明 |
|:-:|------|------|
| 5.1 | 🔗 树莓派 ↔ ESP32 联动 | 树莓派人脸跟随 → 串口发送 `!MOTOR` → ESP32 执行 |
| 5.2 | 🎯 人脸跟随完整测试 | 人在前面走，机器人自动跟随，OLED 显示 🎯 |
| 5.3 | 🗣️ 语音 → 动作联动 | "左转" → STT → 树莓派 → ESP32 → 电机左转 |
| 5.4 | 📟 遥控 → 树莓派联动 | 遥控按 8 → ESP32 → 树莓派 → 启动人脸跟随 |
| 5.5 | 📦 送货场景 | "送水给我" → 语音识别 → 人脸跟随 → 到达 → 停止 → TTS |
| 5.6 | 🧊 防洒测试 | 急刹时 MPU6050 → 托盘舵机反向补偿 |
| 5.7 | 🔄 三模切换 | 遥控模式 ↔ 跟随模式 ↔ 送货模式，切换流畅 |
| 5.8 | 🧪 全场景端到端测试 | 完整用户旅程：说指令 → 机器人执行 → 语音反馈 |

### ✅ 完成标准

- [ ] 说"看着我" → 机器人跟着你走
- [ ] 说"左转" → 机器人原地左转
- [ ] 说"送水给我" → 机器人找到你 → 停在你面前 → 说"水送到了"
- [ ] 按遥控器方向键 → 机器人立即听从遥控
- [ ] 急刹车时托盘上的水不洒出
- [ ] 整个流程从语音指令到执行完成 < 5 秒

## 📊 各部分依赖关系

```
Part 1 (硬件平台)
    ↓
Part 2 (执行器固件)
    ↓
┌───┴───┐
↓       ↓
Part 3  Part 4
(遥控)  (AI核心)
    ↓       ↓
└───┬───┘
    ↓
Part 5 (系统整合)
```

| 部分 | 依赖 | 可以并行？ |
|------|------|:---------:|
| Part 1 | 无 | — |
| Part 2 | Part 1 | ❌ 必须等硬件 |
| Part 3 | Part 2 | ✅ 可以和 Part 4 并行 |
| Part 4 | Part 1（摄像头+音频） | ✅ 可以和 Part 3 并行 |
| Part 5 | Part 2 + Part 3 + Part 4 | ❌ 必须等前三部分 |

---

## 🎯 建议的执行顺序

| 阶段 | 做什么 | 为什么 |
|:---:|------|--------|
| **第 1 步** | Part 1 硬件平台 | 有了物理平台，后面才有东西可以调 |
| **第 2 步** | Part 2 执行器固件 | 所有硬件"活"起来，能通过串口控制 |
| **第 3 步** | Part 3 + Part 4 并行 | 遥控和 AI 可以分开同时做，互不干扰 |
| **第 4 步** | Part 5 系统整合 | 所有模块就位后统一联调 |
